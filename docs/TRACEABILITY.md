## Непроверенная трассировка

- Full production drift v4 window was not accumulated on the operator PostgreSQL/Bybit environment.
- Feature/probability PSI still uses stored published signals; an immutable bounded all-opportunity telemetry ledger for evaluated no-trade symbol opportunities is not implemented.
- Correct terminal coverage and actionability denominators prevent false retry/block/quarantine but do not establish profitability.

- Full retraining and forward validation of the v25 sparse-jackknife interaction evidence were not executed against the operator PostgreSQL/Bybit environment.
- The interaction partition is statistical, not causal; five trades are only a safety floor. The jackknife removes dependence on one tiny cell but does not estimate every tiny cell reliably.
- The regime threshold and interaction cells have not been forward-validated on the operator data.

- Full retraining, candidate promotion and runtime publication under entry schema v2 were not executed against the operator PostgreSQL/Bybit environment.
- Exact historical bid/ask, orderbook queue, operator reaction latency and sub-hour gap path remain unavailable; next-hour open plus directional half-spread is still a proxy.
- Stricter anchoring can reduce trade count; it prevents geometry drift but does not establish positive expected value or explain historical manual-fill losses causally.

- Live decision-time account/orderbook/ticker refresh was not executed against the operator PostgreSQL/Bybit environment; unit tests verify ordering and fail-closed behavior.
- Full-universe orderbook refresh wall-clock duration and rate-limit behavior were not benchmarked.
- Dynamic `4/1206` readiness remains prospective by design; pre-ledger membership/spread evidence is not reconstructed.

- Actual PostgreSQL background fit with a frozen trigger profile was not executed because no isolated integration database was configured.
- Stage-by-stage pre-fit loss across continuity, market context and labels remains diagnostic work; post-fit quality gate now blocks material scope divergence.
- Reproducible training scope does not establish profitability or explain historical manual fill losses causally.

- Live PostgreSQL as-of replay and actual trainer/backtest run were not executed because no isolated integration database was configured.
- Pre-ledger dynamic membership, static-mode historical spread cohorts, orderbook depth and realized fill probability are not reconstructed.
- Correct cohort alignment does not prove profitability and may reduce available training rows.

- PostgreSQL migration `0017`, BYTEA round-trip and immutable trigger were not executed because no isolated integration database was configured.
- Actual operator release replacement and runtime restoration were not performed.
- A pre-1.36.0 artifact already missing from every filesystem/backup cannot be recovered by the new migration.
- Database/backup growth and restore latency with many near-limit artifacts were not benchmarked.

- Live PostgreSQL execution of the latest-prior ticker query and query-plan performance were not run because an isolated integration database is unavailable.
- Live decision-time ticker refresh was not executed against the operator PostgreSQL/Bybit environment; unit tests verify ordering, fail-closed behavior and diagnostics.

- Actual PostgreSQL execution/performance of outcome ID batch queries was not run because an isolated integration database is unavailable.
- The report is descriptive. Dependence-aware confidence intervals, causal treatment effects and attribution of the operator's actual manual fill PnL are not implemented.
- Plans with `NOT_SIZED`, `FUNDING_UNAVAILABLE`, `PATH_UNAVAILABLE` or `INVALID_INPUT` have no invented `counterfactual_r`; only TP/SL/TIMEOUT counts remain available when signal outcomes mature.
- PostgreSQL integration tests, actual migration/index execution and `EXPLAIN ANALYZE` were not run because an isolated PostgreSQL URL/server is unavailable.
- Structural row reduction and streaming are tested, but wall-clock latency and peak RSS on a production-size multi-year ledger were not profiled.
- Evidence before the first successful 1.29.0 snapshot cannot be reconstructed and is intentionally excluded.
- The worker timestamp is local UTC receipt/commit evidence, not an exchange-native historical universe timestamp.
- Exact historical orderbook, queue position, partial fills, latency, min-order rules and operator state remain outside this work package.
- Exact cohort reconstruction does not establish economic edge or explain past losses causally.

- Windows `CREATE_NEW_PROCESS_GROUP`/`taskkill /T /F` runtime was not executed on a Windows host; only the command/branch contract was unit-tested.
- POSIX descendants that deliberately detach into a new session/process group are outside this release's group-containment guarantee.
