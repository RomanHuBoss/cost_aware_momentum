# Patch 1.38.0 — immutable trainer preflight scope

Date: 2026-07-07

## Problem

`BackgroundTrainer.due_reason()` built a dynamic preflight profile with `AUTO_TRAIN_MAX_SYMBOLS` and persisted the exact replay-eligible symbols in `training_data_profile`. The actual `run_training_once()` path reused that list only in static mode. In dynamic mode it called `load_training_market_data(symbols=None, max_symbols=0)`, which loaded every available symbol and selected the newest database horizon.

Therefore the dataset that authorized training could differ from the dataset that was fitted:

- preflight used a bounded historical cohort;
- fit could add symbols outside the preflight cohort;
- fit could include candles and universe snapshots committed after preflight;
- post-feature/context/label filtering could remove a material share of expected symbols without a dedicated coverage reason in the quality gate.

This mismatch could change class balance, actionability density, walk-forward folds, policy trade rate and candidate/incumbent comparison after the scheduler had already declared the input data sufficient.

## Correction

- Added `require_training_trigger_profile()` as a fail-closed background-training boundary.
- Both static and dynamic background training now require a valid trigger `training_data_profile` with non-empty exact symbols and a label cutoff.
- Actual loading always receives the exact preflight symbols and `max_symbols=0`; no second dynamic cohort selection is performed.
- `load_training_market_data()` accepts `maximum_open_time` and applies it to last, mark and index candles.
- The background upper bound is deterministically derived as `preflight_profile.end_time + horizon_hours`, retaining the future candles required for labels while excluding data arriving after preflight.
- `evaluate_quality_gate()` can receive the expected preflight profile and rejects:
  - changed symbol scope;
  - a different minimum-rows contract;
  - fitted symbol coverage below `AUTO_TRAIN_MIN_SYMBOL_COVERAGE_RATIO`;
  - a missing fitted end time;
  - fitted data advancing beyond the preflight cutoff.
- Gate evidence now contains both expected and actual training profiles.

## Compatibility

- Database migration: none. Alembic head remains `0017_model_artifact_blobs`.
- New `.env` variables: none.
- Active artifacts and inference behavior are unchanged.
- Manual training remains compatible because the new quality-gate profile argument is optional and the loader upper bound defaults to `None`.
- Background attempts require the existing trigger profile already generated by `due_reason()` and operator recovery.
- No quality, holdout, walk-forward, minimum trade-rate, spread, EV/RR, leverage or risk threshold is relaxed.

## Operator action

1. Stop the trainer process.
2. Replace application files; no migration or `.env` edit is required.
3. Restart trainer.
4. Allow the scheduler to create a fresh preflight trigger and candidate.
5. Inspect the candidate quality gate `absolute.training_scope` section if a candidate remains inactive.

This patch makes background training reproducible relative to the scheduler decision. It does not increase signal frequency, weaken safeguards or establish profitability.
